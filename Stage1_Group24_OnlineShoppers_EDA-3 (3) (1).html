1.3 Data Description

  • The Online Shoppers Purchasing Intention dataset contains 12,330 web sessions, each representing a unique user over a one-year period, with 18 variables describing user behavior and whether a purchase (Revenue) occurred.
  • After excluding sessions from Region == 1 (per group rule), the dataset contains 7,550 observations and 18 variables.
  • The dataset includes 10 numerical features (e.g., Administrative, Informational, ProductRelated, BounceRates, ExitRates, PageValues, SpecialDay, etc.) and 8 categorical/binary features (e.g., Month, VisitorType, Weekend, Region, OperatingSystems, Browser, TrafficType, and Revenue).
  • There are no missing values, and the target variable Revenue is imbalanced (~15.5% positive, 84.5% negative).
  • The data capture behavioral signals such as time spent on pages (_Duration), engagement (BounceRates, ExitRates), and contextual factors (Month, VisitorType, Weekend).
  • Each row corresponds to a single user session, ensuring no duplication or temporal bias.

Section 4: Method and Plan

  • Planned method. I plan to fit a logistic regression model with Revenue (purchase vs no purchase) as the binary response, using engagement variables such as PageValues, ExitRates, BounceRates, ProductRelated, and the corresponding duration counts, along with categorical predictors like VisitorType, Weekend, and Month.

  • Why this method is appropriate. Logistic regression is designed for binary outcomes and lets me estimate how each predictor is associated with the log-odds of a purchase, which matches my goal of identifying factors related to purchase intention and their direction and relative strength.

  • Key assumptions. The model assumes that individual sessions are independent and that the log-odds of purchase change approximately linearly with each continuous predictor. It also assumes no extreme multicollinearity among predictors and that there is no complete separation of the outcome by any single variable.

  • Potential limitations or weaknesses. Logistic regression only captures linear effects on the log-odds scale and may miss complex non-linear relationships or interactions between variables. Class imbalance (only a minority of sessions end in a purchase) can lead to poor calibration or biased classification unless we adjust the model. In addition, unmeasured factors (such as marketing campaigns or user history) may confound some associations, so results should be interpreted as associations rather than causal effects.

from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score

# -----------------------------
# 1. Design matrix and outcome
# -----------------------------
# Numeric predictors (engagement / intent variables)
num_features = [
    "PageValues",
    "ExitRates",
    "BounceRates",
    "ProductRelated",
    "ProductRelated_Duration",
    "Administrative",
    "Administrative_Duration",
    "Informational",
    "Informational_Duration",
    "SpecialDay"
]

# Categorical predictors
cat_features = ["VisitorType", "Weekend", "Month", "TrafficType"]

# Subset and one-hot encode categorical variables
X_num = df[num_features].copy()
X_cat = pd.get_dummies(df[cat_features], drop_first=True)

X = pd.concat([X_num, X_cat], axis=1)
y = df["Revenue"].astype(int)  # 1 = purchase, 0 = no purchase

# -----------------------------
# 2. Train/test split + scaling
# -----------------------------
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.2, random_state=123, stratify=y
)

scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled = scaler.transform(X_test)

# -----------------------------
# 3. Fit logistic regression
# -----------------------------
log_reg = LogisticRegression(
    max_iter=1000,
    class_weight="balanced",   # help with class imbalance
    solver="lbfgs"
)
log_reg.fit(X_train_scaled, y_train)

# -----------------------------
# 4. Coefficient + odds-ratio table
# -----------------------------
coef = log_reg.coef_[0]

coef_table = (
    pd.DataFrame({
        "Predictor": X.columns,
        "Coef_log_odds": coef,
        "Odds_ratio": np.exp(coef),
    })
    .assign(
        Abs_log_odds=lambda d: d["Coef_log_odds"].abs()
    )
    .sort_values(by="Abs_log_odds", ascending=False)
)

# Show only one table (top 12 predictors by |log-odds|, i.e. most impactful)
print(coef_table[["Predictor", "Coef_log_odds", "Odds_ratio"]].head(12))

# Generate predictions on the scaled test set and compute the classification accuracy
y_pred = log_reg.predict(X_test_scaled)
test_acc = accuracy_score(y_test, y_pred)
print(f"Test accuracy: {test_acc:.3f}")
